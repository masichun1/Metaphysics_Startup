# Domestic_XHS_Project — 国内小红书业务主控

## 核心目标
国内小红书玄学赛道自动化运营，采用 **"Claude 为大脑，OpenCLI 为四肢"** 的闭环架构。

## 运营平台
小红书（国内），中文内容。

## 架构逻辑
```
┌──────────────────────────────────────┐
│           Claude (大脑)              │
│   分析数据、制定策略、生成脚本/话术    │
└──────────┬───────────────────────────┘
           │ 策略与脚本下发
           ▼
┌──────────────────────────────────────┐
│         OpenCLI (四肢/眼睛)          │
│   读取网页、监控数据、调研竞品、       │
│   自动执行发布/互动等操作             │
└──────────┬───────────────────────────┘
           │ 生数据回传
           ▼
┌──────────────────────────────────────┐
│           Claude (大脑)              │
│   分析回传数据，迭代优化策略          │
└──────────────────────────────────────┘
```

## 安全红线
小红书风控极严，所有 OpenCLI 脚本设计必须重点考虑防爬虫、防封号机制：
- Playwright 模拟真人操作（鼠标轨迹、滚动行为、打字节奏）
- 指纹浏览器（Browser Fingerprint 伪装）
- 随机休眠间隔（避免固定频率触发风控）
- 代理 IP 池轮换
- Cookie 与 Session 管理
- 请求频率控制与熔断机制
