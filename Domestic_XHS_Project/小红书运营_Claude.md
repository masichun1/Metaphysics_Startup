# 小红书运营 — OpenCLI × Claude 自动化工作流

## 业务目标
国内小红书玄学赛道自动化运营。

## 架构逻辑
采用 **"OpenCLI 作为四肢/眼睛，Claude 作为大脑"** 的闭环架构：

- **OpenCLI** 负责读取、监控网页和调研竞品，并将生数据传回给 Claude。
- **Claude** 负责分析数据、制定策略，生成新的执行脚本或话术，发回给 OpenCLI 去自动执行。

## 核心开发任务（Skills）

### Skill 01 — 数据自动追踪
- **目标**：定时抓取指定关键词或对标账号的数据表现。
- **数据维度**：笔记点赞数、收藏数、评论数、分享数、粉丝增长趋势、发布时间、文案字数、标签使用。
- **技术实现**：OpenCLI 定时爬取搜索结果页/个人主页，结构化存储至本地数据库或 CSV，Claude 分析趋势并输出优化建议。

### Skill 02 — 评论区监控
- **目标**：实时分析评论区情绪，提取高频问题或潜在客户意向线索。
- **数据维度**：评论情感倾向（正面/负面/中性）、高频关键词、用户提问聚类、竞品评论区导流信号。
- **技术实现**：OpenCLI 爬取指定笔记的评论列表，Claude 进行 NLP 情感分析和意图识别，输出潜在客户名单与应对话术。

### Skill 03 — 竞品/同行调研
- **目标**：爬取并拆解爆款图文的结构、文案逻辑和标签。
- **数据维度**：标题公式、封面风格、正文结构、互动话术、标签组合、发布时间规律、评论区运营策略。
- **技术实现**：OpenCLI 爬取对标账号/话题下的高互动笔记，Claude 逆向拆解爆款公式，生成本号可复用的内容模板。

### Skill 04 — 新号冷启动辅助
- **目标**：生成高点击率的起号选题库和文案框架。
- **核心策略**：
  - 基于 Skill 03 的竞品分析结果，提炼当前赛道的爆款选题方向。
  - 结合玄学赛道特性（神秘感、情感共鸣、实用性），生成差异化选题。
  - 提供多版本文案框架（悬念型、干货型、情感型、争议型），适配不同内容风格测试。
- **技术实现**：Claude 基于竞品数据生成选题库和文案模板，OpenCLI 可辅助批量创建草稿笔记。

### Skill 05 — 引导关注
- **目标**：根据不同场景，生成自然且高转化的引流话术策略。
- **场景覆盖**：
  - 评论区互动 → 引导关注话术
  - 私信回复 → 粉丝转化话术
  - 笔记结尾 CTA → 关注/收藏引导
  - 直播话术 → 实时互动转化
- **技术实现**：Claude 生成多版本话术库，OpenCLI 辅助自动回复（需谨慎控制频率，避免触发风控）。

## 安全红线

小红书风控极严，所有 OpenCLI 脚本设计必须重点考虑防爬虫、防封号机制：

| 防护维度 | 具体措施 |
|---------|---------|
| **浏览器模拟** | 使用 Playwright 模拟真人操作：鼠标轨迹、滚动行为、打字节奏 |
| **指纹伪装** | 指纹浏览器（Browser Fingerprint 伪装），Canvas/WebGL/Audio 指纹随机化 |
| **时序控制** | 随机休眠间隔（500ms-5s 不等），避免固定频率触发风控 |
| **IP 策略** | 代理 IP 池轮换，单 IP 每日请求上限控制 |
| **会话管理** | Cookie 与 Session 持久化管理，模拟正常用户登录态 |
| **熔断机制** | 请求频率实时监控，触发验证码/异常响应时自动降级暂停 |
| **账号隔离** | 一机一号一 IP，账号间操作完全隔离 |
